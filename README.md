# 🚀 Gaming Zone App

A web-based game management application built with ASP.NET Core MVC, .NET 8, and SQL Server.  
Users can create, edit, and delete their own games, as well as add other users' games to a personalized favorites list.  
The app features role-based access control (Admin, Moderator, User), SEO-friendly slug URLs, pagination, search, soft-delete, toast notifications, and dedicated admin/moderator dashboards.

## 📖 About the Project

This project is a web-based Game Management application built with ASP.NET Core MVC on .NET 8, using SQL Server as the database.

The application allows users to manage their own game collection by creating, editing, and deleting games.  
Users can browse games created by others and add them to a personalized favorites list for easy access.

The project follows a **multi-layer architecture** with separate class library projects for Data, Models, Services, ViewModels, Infrastructure, and a shared Common library. It uses the **Repository Pattern** for data access, a dedicated **Service Layer** for business logic, and **Dependency Injection** throughout.

Key architectural highlights:
- **Role-based authorization** with three roles: Admin, Moderator, and User — each with dedicated MVC Areas and dashboards.
- **Soft-delete** for games via an `IsDeleted` flag and EF Core global query filters.
- **SEO-friendly slug URLs** generated by a custom `SlugGenerator` utility.
- **Pagination** using a generic `PaginatedList<T>` class for all list views.
- **Search functionality** across game titles, genres, developers, and publishers.
- **Toast notifications** via Toastr, driven by TempData messages.
- **Identity seeding** at startup for roles, an admin user, and a moderator user via custom middleware extensions.
- **Custom error pages** for 400, 403, 404, and 500 status codes.
- **Unit tests** with NUnit, Moq, and MockQueryable covering services and controllers.

## ✨ Features

### Authentication & Authorization
- [x] **User registration and login** — ASP.NET Core Identity with custom `ApplicationUser` (Guid-based).
- [x] **Role-based access control** — Three roles (`Admin`, `Moderator`, `User`) with dedicated MVC Areas and dashboards.
- [x] **Identity seeding at startup** — Roles, admin user, and moderator user seeded automatically via custom middleware extensions (`UseRolesSeeder`, `UseAdminUserSeeder`, `UseModeratorUserSeeder`).
- [x] **Configurable Identity options** — Password policy, lockout, sign-in settings loaded from `appsettings.Development.json`.

### Game Management
- [x] **Full CRUD for games** — Create, Read, Update, Delete with creator ownership enforcement.
- [x] **Soft-delete** — Games marked via `IsDeleted` flag; EF Core global query filter auto-excludes them from queries.
- [x] **Favorites system** — Many-to-many relationship (`ApplicationUserGame` join entity); add/remove games to a personal favorites list with validation (can't favorite own games).
- [x] **Search** — Search across game titles, genres, developers, and publishers on the All Games page.
- [x] **Pagination** — Generic `PaginatedList<T>` used across all listing views (games, developers, publishers, admin panels).

### Developer & Publisher Management
- [x] **Developer/Publisher browsing** — Public paginated listings with game counts and logo images.
- [x] **Developer/Publisher CRUD** — Admin and Moderator areas for creating, editing, and deleting developers and publishers.

### Admin Dashboard
- [x] **Game management** — Edit and delete any game regardless of creator.
- [x] **Developer & Publisher management** — Full CRUD operations.
- [x] **User management** — List all users with their roles, assign/remove roles, delete users.

### Moderator Dashboard
- [x] **Game management** — Edit and delete any game.
- [x] **Developer & Publisher management** — Edit and delete operations.

### UI & UX
- [x] **Responsive design** — Bootstrap 5 layout with Bootstrap Icons.
- [x] **Toast notifications** — Toastr library driven by TempData (`Success`, `Error`, `Warning`, `Info`).
- [x] **SEO-friendly slug URLs** — Custom `SlugGenerator` creates URL slugs for game detail pages.
- [x] **Developer & Publisher logos** — Entity images displayed on game detail and listing pages.
- [x] **Star ratings** — Visual star display on game detail pages.
- [x] **Custom error pages** — Dedicated views for 400, 403, 404, and 500 via `UseStatusCodePagesWithRedirects`.
- [x] **Reusable partials** — `_AllGamesPartial`, `_PaginationPartial`, `_NotificationPartial`, `_LoginPartial`.

### Architecture & Code Quality
- [x] **Multi-layer solution** — 8 projects organized in 4 solution folders (Web, Data, Services, Tests).
- [x] **Repository pattern** — Dedicated repository interfaces and implementations for all entities.
- [x] **Service layer** — Business logic separated from controllers with clear interface contracts.
- [x] **DTOs** — Separate `Services.Models` project for data transfer between repository and service layers.
- [x] **Centralized constants** — Validation rules, exception messages, output messages, and app settings in `GCommon`.
- [x] **Input validation** — Server-side (data annotations) + client-side (jQuery Validation).
- [x] **Anti-forgery tokens** — `[AutoValidateAntiforgeryToken]` on all base controllers.
- [x] **Unit tests** — NUnit + Moq + MockQueryable covering services and controllers.
- [x] **Seed data** — Pre-seeded games, developers, publishers (with AI-generated images), roles, and admin/moderator users.

## 💻 Usage

```
Follow these steps after the app is running locally to use the main features.

Prerequisites
- Ensure the database is created/migrated: run `dotnet ef database update` (or use Visual Studio __Package Manager Console__ with __Update-Database__).
- Start the app: `dotnet run` or press Run in Visual Studio. Open the URL shown in the console (e.g. `https://localhost:7038`).

Authentication
1. Register a new account: navigate to `/Identity/Account/Register`.
2. Log in: navigate to `/Identity/Account/Login`.

Seeded admin (development)
- Username: `admin`  
- Email: `admin@example.com`  
- Password: `Admin123!`  
(You can register another account for testing the functionality.)

Core workflows
1. Browse games
   - Open `GET /Games` to see the catalog. Click a title to open `GET /Games/GameDetails?id={gameId}`.

2. Add a game (authenticated)
   - Go to `GET /Games/AddGame`.
   - Fill the form: `Title`, `ReleaseDate`, `Genre` (select), `Description`, `ImageUrl` (absolute URL or site path starting with `/images/...`), `Developer` (dropdown), `Publisher` (dropdown).
   - Submit the form (POST to `/Games/AddGame`). New games are associated with your account.

3. Edit / Delete a game (creator only)
   - On a game details page, use `Edit` (`GET /Games/EditGame?id={gameId}` / `POST /Games/EditGame`) or `Delete` (`GET /Games/DeleteGame?id={gameId}` / `POST /Games/DeleteGame`). The app enforces that only the creator can edit or delete.

4. My Games & Favorites
   - `GET /Games/MyGames` — list games you created.
   - `GET /Games/MyFavoriteGames` — list games you added to favorites.
   - Add to favorites (POST) — action invoked from game details (controller handles `AddToFavorites` and `RemoveFromFavorites`).

5. Developers & Publishers
   - `GET /Developers` — list of seeded developers. Click a developer to view their games (`GET /Developers/DeveloperGames?developerId={id}`).
   - `GET /Publishers` — list of seeded publishers. Click a publisher games (`GET /Publishers/PublisherGames?publisherId={id}`).

Admin / Moderator features (authenticated)
- Admin dashboard: `GET /Admin/Dashboard`
- Manage users, roles, and game submissions
- View and restore deleted games

Notes & tips
- Navigation links for authenticated features (My Games / Favorites / Add Game) appear in the nav when signed in.
- Image files live under `wwwroot/images/`. When seeding, `ImageUrl` values commonly use site-root paths such as `/images/game/gameDefault.png`.
- If you change seed data, create a migration via __Add-Migration__ and apply it via __Update-Database__ (or use the `dotnet ef` CLI).
- If the favicon or other static assets don't update, hard-refresh the browser (Ctrl+F5) or open an Incognito/private window.

Troubleshooting common issues
- If you encounter a `500 Internal Server Error`, check the server logs for stack traces (check the Output window in Visual Studio).
- For database-related issues, ensure your connection string in `appsettings.Development.json` is correct and that SQL Server is running.
```

## 🗄️ Database Setup

The project uses **Entity Framework Core 8.0.22** with a **Code-First** approach and the **Repository Pattern**.

Connection string is configured in `appsettings.Development.json`:

```json
"ConnectionStrings": {
  "DefaultConnection": "Server=.;Database=GamingZoneApp;Trusted_Connection=True;Encrypt=False"
}
```

To create and seed the database:

```bash
dotnet ef migrations add InitialCreate
dotnet ef database update
```
